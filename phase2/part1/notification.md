# 通知需求

## 第一期项目中的待办事项中的工作流中，需要将状态作为通知，通知到申请人和审批人
* 将这些状态通知，整合到“我的通知”中，以及App的消息推送
* 通知人
  * 申请人
  * 审批人
    * 根据流程需要：上级，下级
    * 与代办事项中的逻辑保持一致
* 通知内容
  * 状态改变的详情
  * 与代办事项中的状态类型保持一致

* 代办事项的类型

| 序号| 事项类型 |
| :--: | :--: |
| 1 | 请假审批 |
| 2 | 加班审批 |
| 3 | 采购审批 |
| 4 | 维修审批 |
| 5 | 咨询反馈 |
| 6 | 维修任务 |
